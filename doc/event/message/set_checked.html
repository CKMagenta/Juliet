EVENT/MESSAGE:SET_CHECKED

메세지 확인 시 호출

  • request payload
    • event / MESSAGE:SET_CHECKED
    • data 
      [ { "type":메세지 타입,
	"msg_hash": 메세지 해쉬
	"user_hash": 유저 해쉬
 } ]
  • response payload
    • 성공 시
      • event / MESSAGE:SET_CHECKED
      • status / 1 (SUCCESS)
      • data 
        [ { "msg": 성공메세지 } ]
    • 인수가 부족할 때
      • event / MESSAGE:SET_CHECKED
      • status / 2 (INSUFFICIENT_ARGUMENTS)
      • data 
        [ { "msg": 에러메세지 } ]
