EVENT/MESSAGE:SET_CHECKED

메세지 확인 시 호출

  • request payload
    • event / MESSAGE:SET_CHECKED
    • data
      • type / 메세지 타입
      • msg_hash / 메세지의 seq 해시
      • user_hash / 유저 해시
  • response payload
    • 성공 시
      • event / MESSAGE:SET_CHECKED
      • status / 1 (SUCCESS)
      • data / "success"
    • 인수가 부족할 때
      • event / MESSAGE:SET_CHECKED
      • status / 2 (INSUFFICIENT_ARGUMENTS)
      • data / "insufficient informations"
